# 웹으로 접속하기 가이드

로컬호스트가 아닌 실제 IP 주소나 도메인으로 접속하는 방법입니다.

## 🚀 빠른 시작

### 방법 1: 자동 설정 (추천)

**이미 설정되어 있습니다!** 별도 설정 없이 바로 사용할 수 있습니다.

1. **서버 컴퓨터에서 백엔드 실행:**
   ```bash
   cd backend
   source venv/bin/activate
   python run.py
   ```

2. **서버 컴퓨터에서 관리자 웹 실행:**
   ```bash
   cd admin-web
   npm start
   ```

3. **IP 주소 확인:**
   - macOS/Linux: `ifconfig | grep "inet " | grep -v 127.0.0.1`
   - Windows: `ipconfig`
   - 예: `192.168.0.100`

4. **다른 컴퓨터에서 접속:**
   ```
   http://서버IP주소:3000
   ```
   예: `http://192.168.0.100:3000`

**자동으로 API 서버 주소가 설정됩니다!**
- `http://192.168.0.100:3000`으로 접속하면 → API는 `http://192.168.0.100:8000` 사용
- `http://localhost:3000`으로 접속하면 → API는 `http://localhost:8000` 사용

### 방법 2: 환경 변수로 수동 설정

특정 API 서버를 사용하고 싶은 경우:

1. **`.env` 파일 생성** (`admin-web/.env`):
   ```env
   REACT_APP_API_URL=http://서버IP주소:8000/api
   ```
   예: `REACT_APP_API_URL=http://192.168.0.100:8000/api`

2. **서버 재시작:**
   ```bash
   npm start
   ```

## 🌐 인터넷을 통한 접속 (고급)

인터넷을 통해 외부에서 접속하려면:

### 옵션 1: 포트 포워딩 (라우터 설정 필요)

1. **라우터에서 포트 포워딩 설정:**
   - 포트 8000 → 서버 컴퓨터 IP:8000
   - 포트 3000 → 서버 컴퓨터 IP:3000

2. **공인 IP 확인:**
   - `https://whatismyipaddress.com` 접속

3. **접속:**
   ```
   http://공인IP주소:3000
   ```

⚠️ **보안 주의사항:**
- 방화벽 설정 필수
- HTTPS 사용 권장
- 강력한 비밀번호 사용
- 기본 비밀번호(`admin123`) 반드시 변경

### 옵션 2: 터널링 서비스 (간단)

**ngrok 사용 예시:**

1. **ngrok 설치 및 실행:**
   ```bash
   # 백엔드 터널
   ngrok http 8000
   
   # 관리자 웹 터널 (다른 터미널)
   ngrok http 3000
   ```

2. **ngrok이 제공하는 URL 사용:**
   - 예: `https://abc123.ngrok.io`

⚠️ **주의:** ngrok 무료 버전은 URL이 매번 변경됩니다.

### 옵션 3: 클라우드 배포

- AWS, Google Cloud, Azure 등 클라우드 서비스 사용
- 도메인 연결 가능
- HTTPS 자동 설정

- [ ] 서버 컴퓨터에서 백엔드 실행
- [ ] 서버 컴퓨터에서 관리자 웹 실행
- [ ] 서버 컴퓨터 IP 주소 확인
- [ ] 다른 컴퓨터에서 `http://서버IP:3000` 접속 테스트
- [ ] 로그인 테스트

### 인터넷 접속 (고급):
- [ ] 포트 포워딩 설정 또는 터널링 서비스 설정
- [ ] 방화벽 설정 확인
- [ ] 보안 설정 확인 (비밀번호 변경 등)
- [ ] HTTPS 설정 (권장)

## 🔒 보안 권장사항

1. **기본 비밀번호 변경**
   - 관리자 계정 관리에서 `admin123` 비밀번호 변경

2. **방화벽 설정**
   - 필요한 포트만 열기
   - 불필요한 포트는 차단

3. **HTTPS 사용** (인터넷 접속 시)
   - SSL 인증서 설치
   - 암호화된 연결 사용

4. **접근 제한**
   - 특정 IP만 허용하도록 설정 가능
   - `backend/app/core/config.py`에서 CORS 설정

## ❓ 문제 해결

### CORS 오류가 발생할 때:
- 백엔드 서버가 `0.0.0.0:8000`에서 실행 중인지 확인
- `backend/app/core/config.py`에서 CORS 설정 확인

### 접속이 안 될 때:
1. **방화벽 확인**
   - 서버 컴퓨터의 방화벽에서 포트 8000, 3000 허용

2. **네트워크 확인**
   - 같은 네트워크에 연결되어 있는지 확인
   - `ping 서버IP주소` 명령어로 연결 확인

3. **서버 실행 확인**
   - 백엔드: `http://서버IP:8000/health` 접속 확인
   - 관리자 웹: `http://서버IP:3000` 접속 확인

## 💡 팁

- **고정 IP 사용:** 서버 컴퓨터의 IP 주소가 변경되지 않도록 네트워크 설정에서 고정 IP를 사용하는 것을 추천합니다.
- **도메인 사용:** IP 주소 대신 도메인을 사용하려면 DNS 설정이 필요합니다.
- **자동 시작:** 서버 컴퓨터가 부팅될 때 자동으로 서버가 실행되도록 설정할 수 있습니다 (시스템 서비스 설정).
